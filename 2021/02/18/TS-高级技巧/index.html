interface & type

TypeScript 中有两种定义接口类型的方法，分别是： interface（接口）、type alias（类型别名），它们之间是能够自身继承、相互继承的：

这两者使用上特别相似，但也稍微有些不一样的地方

1. 写法不同

目前项目中使用面向对象的写法会多一些，因此 类、实例 的出现比例会打一些，对于单独 .ts 文件内声明的函数可用 type 做声明会更方便一些
